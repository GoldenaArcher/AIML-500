# Reflection Drafter

## Overview

Use this skill for private reflection drafts tied to coursework or portfolio artifacts. The output should sound like a real student reflection: short, first-person, and based on the actual artifact work rather than generic template wording.

## Workflow

1. Identify the course, assignment, and artifact from the repo and current conversation.
2. Keep reflection content out of public portfolio pages.
3. Store drafts under `reflection/<course>/` with one Markdown file per artifact or assignment.
4. For AIML-501, use `reflection/aiml-501/assignment-<n>-<artifact-name>.md`.
5. Draft the reflection as 2-3 short paragraphs unless the assignment explicitly requires labeled sections.
6. Make sure the draft still covers:
- audience customization
- lessons learned
- feedback and revisions

## Writing Style

- Keep the tone direct and human-sounding.
- Prefer simple wording over polished academic phrasing.
- Include small concrete details from the actual work.
- Avoid repeating rubric language.
- Avoid perfectly balanced, template-like paragraph structure.
- If there was no actual instructor or peer feedback, do not mention the absence of feedback; just describe the revisions made.
- Prefer real first-person perspective, including the user's own preferences, habits, or reasoning, over detached summary language.

## When Not To Use

Do not use this skill for public portfolio pages, value propositions, or polished professional bios. This skill is only for private reflection drafting and revision.
